## 📌 PRÉAMBULE — INSTRUCTIONS DE TRAVAIL POUR CLAUDE CODE

### Règles d'or — à respecter SYSTÉMATIQUEMENT

1. **Explore avant d'agir** : lis `README.md`, `*.csproj`, `package.json`, parcours la structure (`src/`, `Controllers/`, `Services/`, `Entities/`), identifie les conventions existantes (naming, patterns, architecture en couches).
2. **Plan first** : utilise `TodoWrite` pour décomposer chaque module en sous-tâches avant la 1ʳᵉ ligne de code.
3. **Un module à la fois** : ne passe JAMAIS au module suivant sans validation explicite de ma part (« OK module suivant »).
4. **Demande, ne devine pas** : si un choix produit/technique est ambigu, pose la question avant d'implémenter.
5. **Respecte l'existant** : si un pattern est déjà en place (ex. Repository, MediatR, ServiceLayer), suis-le. Si tu introduis un nouveau pattern, justifie-le explicitement.
6. **Commits Git** : Conventional Commits (`feat(b2b): ajout taux remise`, `fix(orders): verrouillage concurrent`), 1 commit logique par sous-tâche.
7. **Migrations EF Core** : nommées clairement (`AddDiscountToB2BClient`), revues avant `dotnet ef database update`. JAMAIS de migration destructive sans confirmation.
8. **Sécurité** : valider TOUTES les permissions côté backend (`[Authorize(Roles="Admin")]`, etc.). Jamais de confiance dans le front.
9. **Pas de breaking change** sur les endpoints existants sans alerte explicite.

### Workflow par module

```
1. Lecture code existant lié (3-5 fichiers max)
2. TodoWrite avec étapes ordonnées
3. Couche DATA  → Entité + Migration EF Core
4. Couche LOGIC → Service + DTO + Validator
5. Couche API   → Controller + Swagger
6. Couche FRONT → API client → Hook → Composant → Page
7. Tests manuels (curl/Postman + UI)
8. Récap des fichiers touchés + DoD checklist cochée
9. Stop → attente validation
```

### Objectif métier

import { useTranslation } from 'react-i18next';

export function ProductCard({ product }) {
  const { t } = useTranslation();
  const { mutate: addToCart, isPending } = useAddToCart();

  const isOutOfStock = product.stock <= 0;

  return (
    <div className="product-card">
      {/* ... visuel existant ... */}
      <button
        onClick={() => addToCart({ productId: product.id, quantity: 1 })}
        disabled={isOutOfStock || isPending}
        className="btn-add-to-cart"
        aria-label={t('product.addToCart', { name: product.name })}
      >
        {isPending ? <Spinner /> : <CartIcon />}

**Store Zustand** :
```js
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

export const useCompareStore = create(
  persist(
    (set, get) => ({
      items: [],
      MAX: 4,
      add: (product) => {
        const items = get().items;
        if (items.length >= get().MAX) return { ok: false, reason: 'MAX' };
        if (items.find(i => i.id === product.id)) return { ok: false, reason: 'DUP' };
        set({ items: [...items, product] });
        return { ok: true };
      },
      remove: (id) => set({ items: get().items.filter(i => i.id !== id) }),
      clear: () => set({ items: [] }),
    }),
    { name: 'compare-store' }
  )
);
```

public async Task SetDefaultAsync(Guid clientId, Guid addressId)
{
    using var tx = await _db.Database.BeginTransactionAsync();
    await _db.ClientAddresses
        .Where(a => a.ClientId == clientId && a.IsDefault)
        .ExecuteUpdateAsync(s => s.SetProperty(a => a.IsDefault, false));

    var target = await _db.ClientAddresses
        .FirstOrDefaultAsync(a => a.Id == addressId && a.ClientId == clientId)
        ?? throw new NotFoundException("Address");
    target.IsDefault = true;
    await _db.SaveChangesAsync();
    await tx.CommitAsync();
}

```csharp
public class OrderCalculatorService
{
    public OrderTotalsDto Compute(Order order, Client client)
    {
        var subtotal = order.Items.Sum(i => i.UnitPrice * i.Quantity);

        decimal discountAmount = 0m;
        decimal? discountRate = null;

        if (client.Type == ClientType.B2B && client.DiscountPercent.HasValue)
        {
            discountRate = client.DiscountPercent.Value;
            discountAmount = Math.Round(subtotal * (discountRate.Value / 100m), 2);
        }

        var total = subtotal - discountAmount;

        return new OrderTotalsDto
        {
            Subtotal = subtotal,
            DiscountRate = discountRate,
            DiscountAmount = discountAmount,
            Total = total
        };
    }
}
```

public async Task<bool> TryLockAsync(Guid orderId, Guid confirmatorId)
{
    var timeout = DateTime.UtcNow.AddMinutes(-5);
    var rows = await _db.Orders
        .Where(o => o.Id == orderId
                 && (o.LockedByConfirmatorId == null
                  || o.LockedAt < timeout
                  || o.LockedByConfirmatorId == confirmatorId))
        .ExecuteUpdateAsync(s => s
            .SetProperty(o => o.LockedByConfirmatorId, confirmatorId)
            .SetProperty(o => o.LockedAt, DateTime.UtcNow));

    if (rows == 0) return false;
    await _hub.Clients.All.SendAsync("OrderLocked", orderId, confirmatorId);
    return true;
}
```

import { DndContext } from '@dnd-kit/core';
import { SortableContext, arrayMove } from '@dnd-kit/sortable';

export function HomepageBuilder() {
  const [blocks, setBlocks] = useState([]);
  const [selectedId, setSelectedId] = useState(null);

  const handleDragEnd = (e) => {
    const { active, over } = e;
    if (active.id !== over?.id) {
      setBlocks(b => arrayMove(b, indexOf(b, active.id), indexOf(b, over.id)));
    }
  };

  return (
    <div className="grid grid-cols-[240px_1fr_320px]">
      <BlockPalette onAdd={(t) => setBlocks([...blocks, makeBlock(t)])} />
      <DndContext onDragEnd={handleDragEnd}>
        <SortableContext items={blocks.map(b => b.id)}>
          <Canvas blocks={blocks} onSelect={setSelectedId} />
        </SortableContext>
      </DndContext>
      <PropsPanel block={blocks.find(b => b.id === selectedId)} onChange={...} />
    </div>
  );
}
```

  duplicateIntent: (id) => api.post(`/admin/chatbot/intents/${id}/duplicate`).then(r => r.data),
  importIntents: (file) => {
    const fd = new FormData();

    return api.post('/admin/chatbot/intents/import', fd, {

export function ConversationThread() {
  const { id } = useParams();
  const { data: conversation } = useConversation(id);
  const { mutate: takeover } = useTakeover();
  const { mutate: sendMsg } = useSendAdminMessage();
  const connection = useSignalR('/hubs/chatbot');

  useEffect(() => {
    if (!connection || !id) return;
    connection.invoke('JoinConversation', id);
    const handler = () => queryClient.invalidateQueries(['conversation', id]);
    connection.on('MessageReceived', handler);
    return () => connection.off('MessageReceived', handler);
  }, [connection, id]);

  const adminId = useAuthStore(s => s.user.id);
  const isMine = conversation?.assignedAdminId === adminId;

  return (
    <div className="flex flex-col h-[calc(100vh-160px)]">
      <Header>
        <h2>Conversation #{id.slice(0, 8)}</h2>
        {!isMine && (
          <Button onClick={() => takeover(id)}>
            {t('conv.takeover')}
          </Button>
        )}
      </Header>

      {isMine && (
        <TakeoverBanner>
          {t('conv.youAreHandlingNow')}
        </TakeoverBanner>
      )}

      <MessageList messages={conversation?.messages || []} />

      {isMine && (
        <AdminInput onSend={(content) => sendMsg({ id, content })} />
      )}
    </div>
  );
}

export function SandboxPanel() {
  const { t } = useTranslation('chatbot');
  const [messages, setMessages] = useState([]);
  const [input, setInput] = useState('');
  const [language, setLanguage] = useState('fr');

  const { mutate: send, isPending } = useMutation({
    mutationFn: ({ message, language }) => chatbotApi.sandboxMessage(message, language),
    onSuccess: (data, vars) => {
      setMessages(m => [
        ...m,
        { sender: 'user', content: vars.message },
        { sender: 'bot', ...data }
      ]);
    }
  });

  const handleSend = () => {
    if (!input.trim()) return;
    send({ message: input, language });
    setInput('');
  };

  return (
    <div className="grid grid-cols-2 gap-6 h-[calc(100vh-160px)]">
      <div className="border rounded-lg flex flex-col">
        <Header>
          {t('sandbox.preview')}
          <Select value={language} onChange={e => setLanguage(e.target.value)}>
            <option value="fr">FR</option>
            <option value="en">EN</option>
            <option value="ar">AR</option>
          </Select>
        </Header>
        <div className="flex-1 overflow-auto p-4 space-y-2">
          {messages.map((m, i) => <Bubble key={i} {...m} />)}
        </div>
        <div className="p-3 border-t flex gap-2">
          <Input value={input} onChange={e => setInput(e.target.value)}
                 onKeyDown={e => e.key === 'Enter' && handleSend()}
                 placeholder={t('sandbox.typeMessage')} />
          <Button onClick={handleSend} disabled={isPending}>

    </div>
  );
}

# 🔐 CONVENTIONS TRANSVERSALES

### Sécurité
- TOUS les endpoints protégés par `[Authorize]` avec rôles explicites.
- Validation FluentValidation sur chaque DTO entrant.
- Aucun champ GPS dans les DTOs admins ou confirmateurs.
- Anti-CSRF activé sur endpoints sensibles.
- Rate-limiting sur endpoints publics critiques (`/api/auth/*`, `/api/chatbot/*`).
- Secrets (LLM keys, JWT secret) jamais commités, jamais retournés en clair.

### Performance
- Pagination obligatoire sur toute liste > 50 items (`?page=&pageSize=`).
- Indexes BDD sur colonnes filtrées (`ClientId`, `Status`, `CreatedAt`, `IntentId`, `ConversationId`, etc.).
- Agrégations stats faites en SQL, pas en mémoire.
- `AsNoTracking()` sur les reads.
- Cache mémoire pour : settings publics, config chatbot, intents actifs (invalidation sur write).

### Logs & Audit
- Serilog : enrichissement avec `UserId`, `CorrelationId`.
- Actions critiques tracées en table d'audit : modif remise B2B, changement état commande, modif homepage, paramétrage app, modif chatbot config, takeover conversation, suppression intent.

### Tests
- Backend : tests d'intégration sur endpoints critiques (xUnit + WebApplicationFactory).
- Frontend : tests composants critiques (Vitest + Testing Library).

### Conventions de nommage
- **Backend** : `PascalCase` classes/méthodes, `_camelCase` champs privés, DTOs suffixés `Dto`, services `IXxxService` / `XxxService`.
- **Frontend** : composants `PascalCase`, hooks `useXxx`, pages `Xxx.jsx`, dossiers `kebab-case`.
- **API** : routes en kebab-case plural (`/api/client-addresses`, `/api/admin/chatbot/intents`).
